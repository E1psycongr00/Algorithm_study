# 숫자 타자 대회

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/150369)

## 문제 요구 사항

### 문제

- 왼손 엄지 4, 오른손 엄지 6
- 가중치
  - 같은 걸 누를 때 1
  - 상하좌우 2
  - 좌상, 우상, 좌하, 우하 3

### 제한 사항

- 주어지는 숫자의 길이 10만
  
## 문제 해결 전략

- 완전 탐색 방식은 시간 초과가 된다. 이것을 동적계획법으로 최적화한다.
  - 완전 탐색을 활용하면 2^숫자 길이(10만) 만큼 탐색을 해야 한다. 여기에 캐싱을 활용해보자.
